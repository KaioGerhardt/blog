O que é e como Funciona a Herança em Java?

Kaio Gerhardt

A Herança no é algo especifico da linguagem Java, mas sim um principio da POO (Programaço Orientada a Objetos) que consequentemente  muito utilizada no mundo Java. Sendo assim um importante tema que todo programador Backend deveria saber, pois uma hora ou outra vai precisar utilizar ou irá se deparar com um código que possui herança.

  • O que é Herança?

A Herança é um principio do mundo OO (Orientado a Objetos) que permite a criação de outras classes apartir de uma especifica. Ou seja, existe uma classe principal denominada super classe e outra classe que herda as caracteristica da classe principal chamada subclasses ou tambem classes derivadas. As subclasses elas erdam tudo da classe principal, desde atributos, metodos e até mesmo o construtor da classe principal. Fazendo com que tudo que for "codado" na classe pai impacte diretamente a classe "filho".

A herança contribui muito para dar manutenção e manter o código, pois assim é possivel que algo seja alterado apenas em um local especifico e outras centenas de classes sofram a mesma alteração. Sem contar da reutilização de códigos, sendo uma prática que demonstra habilidades por parte do programador.

  • Exemplo da utilização de Herança em Java.

Bom, para fins demonstrativos vou fazer um pequeno código simulando um banco, tendo o desenvolvimento de tres classes, a classe Conta e ContaCorrente.

Código da classe Conta:

public class Conta {
    
    private String nome;
    private String cpf;
    private int agencia;
    private int numeroConta;
    private double saldo;
    
    public Conta(String nome, String cpf, int agencia, int numeroconta) {
        this.nome = nome;
        this.cpf = cpf;
        this.agencia = agencia;
        this.numeroConta = numeroconta;
    }
    
    public void sacar(double valor) {
        this.saldo -= valor;
    }
    
    public void depositar(double valor) {
        this.saldo += valor;
    }
}

Bom, desta forma temos a classe principal, ou superclasse, como quiser chamar. Isso quer dizer que outras classes iro herdar as caracteristicas desta classe, ou seja, vo ter os mesmos atributos,mesmos métodos e até mesmo os mesmos construtores (lembrando que é possivel ter mais de um). Mas vamos para o código da outra classe.

Código da classe ContaCorrente:

public class ContaCorrente extends Conta{

    public ContaCorrente(String nome, String cpf, int agencia, int numeroconta) {
        super(nome, cpf, agencia, numeroconta);
    }

}

No código acima tenho a mesma estrutura da classe Conta mas com menos da metade de linhas, fazendo com que toda a classe tenha a mesma estrutura e comportamento. De todo modo, foi utilizado o construtor da classe conta tambem, sendo ele chamado pela linha de cdigo super(nome, cpf, agencia, numeroconta); , a palavra "super" está relacionada a superclasse, portanto passa os mesmos parametro do construtor da classe conta.

Com esta mesma estrutura poderiamos utilizar com diversas outras classes, lembrando que em Java não existe herança multipla, ou seja, uma classe filha só consegue herdar uma classe pai. Já em outras linguagens, como Python, é possivel uma classe filha herdar caracteristicas de mais de uma classe. Com isso, abrimos um leque para outro assunto que é o polimorfismo, mas isto é assunto para outro post.
